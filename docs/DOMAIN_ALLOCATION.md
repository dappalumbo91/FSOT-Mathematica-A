# Domain allocation + D_eff (occupation spaces)

Different interactive systems can share **similar numeric magnitudes** of `S` or token scores while living in **completely different occupation and interaction spaces** — like organs in one organism, or cells that do not rewrite the whole body every tick.

## Rules

1. **One medium** — seeds + `S = K(T1+T2+T3)`.
2. **Many organs** — domains with folds `(D_eff, δψ, observed)`.
3. **Allocation** — tokens occupy domains (curriculum targets solidify membership).
4. **D_eff spine** — embeddings are built around that domain’s D_eff, not a flat global soup.
5. **Local candidates** — scoring only inside the active organ (+ universal bridges), frequency-capped (linguistic diet ≤ 120).
6. **Multi-cue routing** — weighted cues; language outranks soft “spacetime” when both fire; true cosmology needs cosmic cues.
7. **Generation** — ban recent tokens + seed-locked D_eff phase wobble so local loops do not stutter forever.

## Same number, different space

A score of ~0.6 in **medical** (D_eff=13) is not the same occupation as ~0.6 in **quantum** (D_eff=6). Magnitude can rhyme; **interaction neighborhood** does not.
